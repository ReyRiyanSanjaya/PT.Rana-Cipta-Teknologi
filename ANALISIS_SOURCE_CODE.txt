1. RINGKASAN EKSEKUTIF
================================================================================

Rana Market adalah platform hybrid POS (Point of Sales) + Marketplace +
Ride-Hailing yang dirancang untuk UMKM di Indonesia. Merupakan super-app
yang menggabungkan fitur Gojek/Grab dengan sistem POS untuk penjual.

FITUR UTAMA:
  - POS Kasir: transaksi cepat, cetak struk Bluetooth, shift management
  - Inventory: stok real-time, varian, stock opname, barcode scanning
  - Laporan: dashboard laba rugi, grafik, riwayat transaksi, ekspor PDF
  - Marketplace: toko online, keranjang, checkout, review/rating
  - Wholesale/Kulakan: B2B marketplace, pricing tier, kredit limit
  - PPOB: pulsa, token listrik, tagihan (Digiflazz)
  - Ride-Hailing: Ride/Send/Food delivery + tracking real-time
  - Wallet Digital: saldo, top-up, withdraw, transfer antar toko
  - Flash Sale: promo flash sale terjadwal
  - Referral: kode referral + reward
  - Community: forum, topic, post, komentar, like
  - Chat: real-time rooms, typing indicator, online presence
  - Blog: CMS lengkap
  - Support: tiket + live chat
  - Notifikasi: WhatsApp + Socket.io
  - Gamifikasi: Snake, Tetris, Match-3, TTS
  - Driver App: registrasi, dashboard, navigasi, wallet
  - Admin Panel: super admin dashboard, monitoring merchant
  - Distributor Portal: B2B distributor/supplier

4.1 SERVER (BACKEND API)
  Entry: server/src/index.js | Port: 4000

  Middleware Pipeline:
    1. CORS (origin: true, credentials: true)
    2. Helmet (security headers, crossOriginResourcePolicy: false)
    3. Compression (Gzip)
    4. Rate Limiting (3000 req/15min per IP)
    5. Body Parser (JSON/URL-encoded, limit 50MB)
    6. Morgan (HTTP logging, dev mode)
    7. Static Files (/uploads)

  API Routes (24 endpoint groups):
    /api/auth           - Login, Register, Token
    /api/reports        - Laporan penjualan, laba rugi
    /api/transactions   - CRUD transaksi POS
    /api/products       - Manajemen produk
    /api/subscriptions  - Langganan & paket
    /api/purchases      - Pembelian/kulakan
    /api/orders         - Order management merchant
    /api/market         - API publik marketplace
    /api/system         - System settings & monitoring
    /api/admin          - Super admin operations
    /api/wallet         - Wallet digital
    /api/inventory      - Manajemen stok
    /api/tickets        - Tiket support
    /api/wholesale      - B2B wholesale
    /api/distributor    - Portal distributor
    /api/ppob           - Produk digital (PPOB)
    /api/blog           - Blog CMS
    /api/referral       - Sistem referral
    /api/community      - Forum komunitas
    /api/chat           - Real-time chat
    /api/contact        - Contact form
    /api/careers        - Lowongan kerja
    /api/driver         - API driver

6. ALUR & MEKANISME KERJA
================================================================================

6.1 ALUR REGISTRASI & MULTI-TENANCY:
  1. User Register -> POST /api/auth/register
  2. Server buat Tenant baru + User sebagai OWNER
  3. Trial 7 hari dimulai otomatis
  4. User tambah Store -> multi-cabang per Tenant
  5. User tambah karyawan (CASHIER, STORE_MANAGER)
  6. Trial habis -> Write diblokir -> harus subscribe

6.2 ALUR TRANSAKSI POS (OFFLINE-FIRST):
  1. Kasir buka POS Mode
  2. Scan barcode / pilih produk manual
  3. Tambah ke keranjang -> hitung subtotal, tax, diskon
  4. Pilih metode bayar (CASH/QRIS/Transfer)
  5. Proses pembayaran -> hitung kembalian
  6. Transaksi disimpan lokal (SQLite) -> offlineId idempotency
  7. Background sync ke server -> POST /api/transactions
  8. Server: kurangi stok, catat CashflowLog, update DailySalesSummary
  9. Socket.io emit ke tenant room -> update dashboard real-time
  10. Cetak struk via Bluetooth thermal printer / browser print

6.3 ALUR MARKETPLACE (BUYER -> MERCHANT -> DRIVER):
  1. Buyer buka app -> lihat toko terdekat (geolokasi)
  2. Pilih toko -> produk -> keranjang
  3. Checkout -> pilih RIDE/SEND/FOOD -> set alamat
  4. ServiceRequest dibuat (SEARCHING)
  5. Socket.io broadcast ke driver_zone
  6. Driver accept (ACCEPTED) -> menuju merchant (ARRIVED)
  7. Pickup barang (IN_TRANSIT) -> live tracking via Socket.io
  8. Buyer lihat driver di peta real-time
  9. Diantar -> COMPLETED -> Rating & Review

6.4 ALUR WHOLESALE/KULAKAN (B2B):
  1. Distributor daftar -> approval Super Admin
  2. Upload produk + pricing tiers
  3. Merchant browse katalog -> cek MOQ -> keranjang
  4. Checkout -> PENDING -> PAID -> PROCESSING -> SHIPPED -> DELIVERED
  5. Credit limit & payment term per merchant

6.5 ALUR PPOB (INTEGRASI DIGIFLAZZ):
  1. Merchant masuk menu PPOB
  2. Pilih kategori + input nomor pelanggan + nominal
  3. Server kirim ke Digiflazz API
  4. PpobTransaction: PENDING -> SUCCESS/FAILED
  5. SN dikirim ke pelanggan -> profit dari selisih harga

6.6 ALUR AUTENTIKASI:
  1. Login -> bcrypt verify -> generate JWT
  2. JWT payload: {userId, tenantId, storeId, role}
  3. Client kirim: Authorization: Bearer <token>
  4. Middleware: auth.js -> role.js -> subscription.js
  5. WebSocket: token via handshake.auth.token

10. KEKURANGAN & REKOMENDASI
================================================================================

  1. FILE TERLALU BESAR (GOD FILES)
     - adminController.js = 85KB, report_screen.dart = 222KB
     - Rekomendasi: pecah menjadi modul kecil per domain

  2. INKONSISTENSI VERSI LIBRARY
     - React 18 vs 19, Vite 5/6/7, Tailwind 3/4
     - Rekomendasi: standardisasi dengan npm workspaces / Turborepo

  3. TIDAK ADA TESTING
     - Tidak ada unit/integration/e2e test terstruktur
     - Rekomendasi: Jest (server), React Testing Library (web), Flutter test

  4. KEAMANAN JWT HARDCODED
     - Fallback: 'super_secret_key_change_in_prod'
     - Rekomendasi: hapus fallback, crash jika env tidak set

  5. TIDAK ADA INPUT VALIDATION
     - Tidak ada Joi/Zod/express-validator
     - Rekomendasi: implementasikan Zod untuk validasi centralized

  6. CORS TERLALU TERBUKA
     - origin: true / origin: "*" pada Socket.io
     - Rekomendasi: whitelist domain berdasarkan environment

  7. DATABASE TANPA ROW-LEVEL SECURITY
     - Isolasi hanya di level aplikasi
     - Rekomendasi: implementasikan PostgreSQL RLS

  8. TIDAK ADA LOGGING TERSTRUKTUR
     - Hanya console.log/error + Morgan
     - Rekomendasi: Winston atau Pino untuk structured logging

  9. MONOREPO TANPA TOOLING
     - Tidak ada npm workspaces / Turborepo / Nx
     - Rekomendasi: adopsi workspace manager

  10. MULTIPLE PRISMA CLIENT INSTANCES
      - new PrismaClient() di beberapa file
      - Rekomendasi: singleton pattern

  11. RATE LIMIT TERLALU TINGGI
      - 3000 req/15min = 200 req/menit untuk semua endpoint
      - Rekomendasi: rate limit per-route

  12. MOUNT ROUTE DUPLIKAT
      - /api/products dimount DUA KALI di index.js
      - Rekomendasi: hapus duplikat

  13. TIDAK ADA CACHING
      - Tidak ada Redis atau in-memory cache
      - Rekomendasi: Redis untuk data sering diakses

  14. CHAT RETENTION TERLALU AGRESIF
      - Dihapus otomatis setelah 24 jam
      - Rekomendasi: configurable retention period
