Arsitektur Sistem Rana Market

Rana Market dibangun dengan arsitektur Multi-Tenant SaaS (Software as a Service) yang mendukung ekosistem hybrid antara Point of Sales (POS) offline-first dan Marketplace online.

Versi Dokumen: 1.2.0
Terakhir Diupdate: 2026-03-31

1. Gambaran Umum Arsitektur

Sistem ini memisahkan tanggung jawab antara penyimpanan data terpusat (Cloud) dan operasional harian di lapangan (Edge/Mobile). Arsitektur ini dirancang untuk skalabilitas tinggi dan keandalan operasional UMKM.

Konsep Utama:

  • Multi-Tenancy: Setiap bisnis (Tenant) memiliki data yang terisolasi secara logis menggunakan tenantId di setiap tabel database.
  • Offline-First POS: Aplikasi kasir dapat bekerja tanpa internet dan akan melakukan sinkronisasi otomatis saat koneksi tersedia.
  • Hybrid Ecosystem: Integrasi antara manajemen stok internal (POS) dengan etalase publik (Marketplace).

2. Komponen Sistem (Sub-Proyek)

Ekosistem Rana Market terdiri dari 7 komponen utama yang saling terhubung:

Komponen Platform Fungsi Utama
Server API Node.js/Express Pusat logika bisnis, autentikasi, sinkronisasi data, dan integrasi pihak ketiga.
Admin Dashboard Web (React) Panel kontrol untuk Super Admin mengelola seluruh tenant, user, dan sistem global.
Merchant Dashboard Web (React) Panel manajemen toko untuk pemilik (Laporan, Stok, Pengaturan Toko).
Distributor Portal Web (React/TS) Platform untuk supplier mengelola produk grosir dan pesanan kulakan.
Seller App Mobile (Flutter) Aplikasi kasir (POS) utama dengan fitur offline-first, cetak struk, dan PPOB.
Buyer App Mobile (Flutter) Aplikasi marketplace untuk pelanggan belanja dari toko-toko terdekat.
Driver App Mobile (Flutter) Aplikasi khusus kurir untuk manajemen pengantaran pesanan marketplace.

3. Tech Stack

  • Backend: Node.js, Express.js, Prisma ORM.
  • Database: PostgreSQL (Penyimpanan Utama).
  • Frontend Web: React.js, Vite, Tailwind CSS.
  • Mobile: Flutter (Dart), Provider/Bloc untuk State Management.
  • Infrastructure: Nginx (Reverse Proxy), PM2 (Process Manager), VPS Ubuntu.
  • Integrasi: Digiflazz (PPOB), Fonnte (WhatsApp Gateway), Socket.io (Real-time).

4. Alur Data & Komunikasi

A. Sinkronisasi POS (Offline-to-Cloud)

Aplikasi Seller menyimpan transaksi di database lokal (SQLite) terlebih dahulu. Background service akan mengirimkan data dalam bentuk batch ke endpoint /api/sync/upload saat internet tersedia. Server kemudian memproses data tersebut secara asinkron untuk menjaga performa.

B. Integrasi B2B (Kulakan)

Merchant dapat melihat katalog dari Distributor Portal melalui menu Kulakan di aplikasi Seller. Pesanan yang dibuat akan masuk ke dashboard Distributor dan mengurangi stok grosir secara real-time.

C. Real-time Notifications

Menggunakan Socket.io untuk mengirimkan notifikasi instan (misal: pesanan baru masuk) dari server ke aplikasi Mobile atau Dashboard Web tanpa perlu refresh halaman.
