Panduan Deployment Server Full Komplit

Panduan langkah demi langkah untuk menyebarkan (deploy) seluruh ekosistem Rana Market ke lingkungan produksi menggunakan VPS (Ubuntu) dan layanan hosting statis.

Versi Dokumen: 1.2.0
Terakhir Diupdate: 2026-03-31

1. Prasyarat Produksi

  • VPS: Minimum 2 vCPU, 4GB RAM (Rekomendasi: Ubuntu 22.04 LTS).
  • Domain: 1 Domain utama (misal: ranamarket.com) dan Subdomain (misal: api.ranamarket.com).
  • SSL: Sertifikat gratis via Let's Encrypt (Certbot).

2. Persiapan Lingkungan VPS

Login ke VPS Anda via SSH dan jalankan perintah berikut:

# 1. Update paket sistem
sudo apt update && sudo apt upgrade -y

# 2. Install Node.js (Versi 20 LTS)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 3. Install Database (PostgreSQL)
sudo apt install -y postgresql postgresql-contrib

# 4. Install Process Manager & Nginx
sudo npm install -g pm2
sudo apt install -y nginx git certbot python3-certbot-nginx

3. Konfigurasi Database Produksi

Gunakan user terpisah untuk aplikasi agar lebih aman:

sudo -i -u postgres
psql

# Di dalam prompt SQL:
CREATE DATABASE rana_market;
CREATE USER rana_admin WITH ENCRYPTED PASSWORD 'password_sangat_rahasia';
GRANT ALL PRIVILEGES ON DATABASE rana_market TO rana_admin;
ALTER DATABASE rana_market OWNER TO rana_admin;
\q
exit

4. Deployment Backend API (Server)

Pusatkan seluruh logika di folder server/.

A. Upload & Instalasi

cd /var/www
sudo git clone https://github.com/username/rana-project.git
cd rana-project/server
npm install

B. Konfigurasi Environment (.env)

Buat file .env produksi:

PORT=4000
NODE_ENV=production

# Database
DATABASE_URL="postgresql://rana_admin:password_sangat_rahasia@localhost:5432/rana_market?schema=public"

# Security
JWT_SECRET="generate_karakter_acak_panjang"

# Layanan Pihak Ketiga (Ubah ke versi Produksi)
DIGIFLAZZ_USERNAME="user_prod"
DIGIFLAZZ_API_KEY="key_prod"
WA_API_KEY="wa_key_prod"

C. Sinkronisasi Skema & Seeding

npx prisma generate
npx prisma migrate deploy
npm run seed:menus
npm run seed:ppob

D. Menjalankan dengan PM2 (Cluster Mode)

Gunakan ecosystem.config.js untuk mengaktifkan mode cluster agar aplikasi lebih stabil.

pm2 start ecosystem.config.js --env production
pm2 save
pm2 startup

5. Konfigurasi Reverse Proxy (Nginx)

Nginx bertugas meneruskan trafik dari internet (port 80/443) ke Node.js (port 4000) dan menangani WebSockets (Socket.io).

sudo nano /etc/nginx/sites-available/rana-api

Isi dengan konfigurasi berikut (sesuaikan server_name):

server {
    listen 80;
    server_name api.ranamarket.com;

    location / {
        proxy_pass http://localhost:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Untuk upload gambar produk yang besar
    client_max_body_size 50M;
}

Aktifkan konfigurasi dan install SSL:

sudo ln -s /etc/nginx/sites-available/rana-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

# Install SSL Let's Encrypt
sudo certbot --nginx -d api.ranamarket.com

6. Deployment Dashboard Web (Front-end)

Terdapat 3 dashboard (Admin, Merchant, Distributor). Disarankan menggunakan layanan seperti Vercel atau Netlify untuk performa terbaik dan integrasi CI/CD otomatis.

Langkah Deployment Front-end:

  1. Hubungkan repositori GitHub ke Vercel.
  2. Tambahkan 3 project baru untuk folder: admin_client, client, dan distributor_client.
  3. Atur Environment Variable di setiap project Vercel: 
    VITE_API_URL=https://api.ranamarket.com/api
  4. Vercel akan secara otomatis melakukan build (npm run build) dan deploy.

7. Go-Live Checklist

  • Environment: Pastikan NODE_ENV=production aktif.
  • Firewall: Aktifkan UFW (Hanya port 22, 80, 443 yang terbuka).
  • Monitoring: Cek PM2 logs secara berkala (pm2 logs).
  • Backup: Atur cronjob untuk backup database harian.
  • API Keys: Pastikan semua kunci Digiflazz/WA menggunakan versi Production.
