Struktur Database (Schema)

Rana Market menggunakan PostgreSQL dengan Prisma ORM untuk mengelola data relasional yang kompleks dalam model Multi-Tenant SaaS.

Versi Dokumen: 1.2.0
Terakhir Diupdate: 2026-03-31

1. Core Tenancy & IAM

Sistem ini dirancang untuk mengisolasi data antar penyewa (Tenant) secara logis.


model Tenant {
  id                 String             @id @default(uuid())
  name               String
  plan               SubscriptionPlan   @default(FREE)
  subscriptionStatus SubscriptionStatus @default(TRIAL)
  subscriptionEndsAt DateTime?
  // Relations: users, stores, products, transactions, etc.
}

model User {
  id           String   @id @default(uuid())
  tenantId     String
  storeId      String?
  name         String
  email        String   @unique
  passwordHash String
  role         UserRole @default(CASHIER)
  // Owner, Admin, StoreManager, Cashier, SuperAdmin, Distributor
}

enum SubscriptionPlan { FREE, BASIC, PREMIUM, ENTERPRISE }

2. Business Data & POS

Data operasional toko, produk, dan transaksi penjualan harian.


model Store {
  id           String   @id @default(uuid())
  tenantId     String
  name         String
  waNumber     String?  @unique
  balance      Float    @default(0) // Saldo Digital Wallet
  latitude     Float?
  longitude    Float?
}

model Product {
  id           String   @id @default(uuid())
  tenantId     String
  storeId      String?
  name         String
  sku          String?
  basePrice    Float
  sellingPrice Float
  stock        Int      @default(0)
}

model Transaction {
  id            String            @id @default(uuid())
  tenantId      String
  storeId       String
  totalAmount   Float             @default(0)
  paymentMethod String            @default("CASH")
  orderStatus   TransactionStatus @default(COMPLETED)
  occurredAt    DateTime          @default(now())
}

3. B2B Wholesale (Kulakan)

Fitur untuk menghubungkan Merchant dengan Supplier/Distributor.


model Distributor {
  id             String         @id @default(uuid())
  userId         String         @unique
  companyName    String
  approvalStatus ApprovalStatus @default(PENDING)
  balance        Float          @default(0)
  wholesaleProducts WholesaleProduct[]
}

model WholesaleProduct {
  id                  String   @id @default(uuid())
  distributorId       String
  name                String
  pricingTiers        Json // Skema harga grosir bertingkat
  moq                 Int      @default(1)
  stockQuantity       Int      @default(0)
}

model WholesaleOrder {
  id              String               @id @default(uuid())
  tenantId        String
  distributorId   String
  orderNumber     String               @unique
  totalAmount     Float
  status          WholesaleOrderStatus @default(PENDING)
}

4. Cash & Finance

Manajemen keuangan, hutang, dan dompet digital toko.


model CashflowLog {
  id          String   @id @default(uuid())
  storeId     String
  type        String?  // IN, OUT
  category    String?  // SALES, EXPENSE, WITHDRAWAL, etc.
  amount      Float
  occurredAt  DateTime @default(now())
}

model Withdrawal {
  id            String           @id @default(uuid())
  storeId       String
  amount        Float
  status        WithdrawalStatus @default(PENDING)
  bankName      String
  accountNumber String
}

model DebtRecord {
  id          String     @id @default(uuid())
  customerId  String
  amount      Float
  status      DebtStatus @default(UNPAID)
}

5. PPOB & Digital Products

Transaksi produk digital melalui integrasi Digiflazz.


model PpobTransaction {
  id           String     @id @default(uuid())
  storeId      String
  refId        String     // ID Referensi dari Digiflazz
  productCode  String
  customerNo   String     // Nomor HP atau ID Pelanggan
  amount       Float      // Harga modal
  sellingPrice Float      // Harga jual ke pelanggan
  status       PpobStatus @default(PENDING)
}

6. Driver & Ride-Hailing

Manajemen kurir dan layanan pengantaran makanan/barang.


model Driver {
  id           String       @id @default(uuid())
  name         String
  phone        String       @unique
  vehiclePlate String
  status       DriverStatus @default(OFFLINE)
  balance      Float        @default(0)
  latitude     Float?
  longitude    Float?
}

model ServiceRequest {
  id         String        @id @default(uuid())
  type       ServiceType   // RIDE, SEND, FOOD
  driverId   String?
  status     ServiceStatus @default(SEARCHING)
  price      Float
  originAddress String
  destAddress   String
}

7. Community & Content

Fitur interaksi antar pengguna dan pengelolaan konten blog/pengumuman.


model CommunityPost {
  id        Int      @id @default(autoincrement())
  topicId   Int
  authorId  String
  title     String?
  content   String
  likes     Int      @default(0)
}

model BlogPost {
  id          String    @id @default(uuid())
  title       String
  slug        String    @unique
  content     String
  status      String    @default("DRAFT")
}

Penjelasan Teknis Tambahan

  • UUID: Sebagian besar tabel menggunakan UUID sebagai Primary Key untuk keamanan dan kemudahan sinkronisasi offline.
  • JSONB: Digunakan untuk menyimpan data dinamis seperti pricingTiers (harga grosir) dan paymentDetails.
  • Enums: Digunakan secara luas untuk menjaga konsistensi status (Order, Subscription, Request).
  • Indexes: Indeks dibuat secara otomatis oleh Prisma pada kolom id, unique, dan relasi kunci asing untuk optimasi query.
