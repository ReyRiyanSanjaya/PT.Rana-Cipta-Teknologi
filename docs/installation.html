Panduan Instalasi Lokal

Panduan lengkap untuk menjalankan seluruh ekosistem Rana Market di komputer lokal Anda, mulai dari backend hingga 3 dashboard web dan 3 aplikasi mobile.

Versi Dokumen: 3.0.0  |  Terakhir Diupdate: 2026-04-07  |  Stack: Node.js 20 · PostgreSQL 14+ · Flutter 3.10+ · React + Vite

1. Prasyarat Sistem

Pastikan semua perangkat lunak berikut sudah terinstall sebelum memulai:

SoftwareVersi MinimumDownload
Node.js v18.x atau v20.x LTS nodejs.org
PostgreSQL v14+ postgresql.org
Flutter SDK v3.10+ flutter.dev
Git Terbaru git-scm.com
Android Studio Terbaru developer.android.com

Verifikasi instalasi dengan membuka CMD dan jalankan:

node --version
flutter --version
git --version

2. Persiapan Sebelum Instalasi

Dua langkah ini WAJIB dilakukan secara manual sebelum menjalankan installer otomatis:

A. Buat Database PostgreSQL

Buka pgAdmin atau SQL Shell (psql), lalu jalankan:

CREATE DATABASE rana_pos;

Atau dari CMD/psql:

psql -U postgres -c "CREATE DATABASE rana_pos;"

B. Konfigurasi File server/.env

Buka file server\.env di text editor dan sesuaikan password PostgreSQL Anda:

PORT=4000
DATABASE_URL="postgresql://postgres:PASSWORD_ANDA@localhost:5432/rana_pos?schema=public"
JWT_SECRET="ganti_dengan_string_acak_panjang_minimal_32_karakter"
NODE_ENV=development
⚠️ Ganti PASSWORD_ANDA dengan password akun postgres yang Anda set saat instalasi PostgreSQL. Pastikan tidak ada teks tambahan seperti PASSWORD_ di depan password sebenarnya.

✅ Contoh benar: DATABASE_URL="postgresql://postgres:simpledark67@localhost:5432/rana_pos?schema=public"
❌ Contoh salah: DATABASE_URL="postgresql://postgres:PASSWORD_simpledark67@localhost:5432/rana_pos?schema=public"

3. Instalasi Otomatis — Install manual awal.bat ⭐

Setelah prasyarat siap, jalankan file berikut dari root folder proyek:

📁 Install manual awal.bat

Double-click file Install manual awal.bat di root folder. Script ini akan melakukan SEMUA langkah secara otomatis:

  1. ✅ Verifikasi Node.js dan Flutter tersedia di sistem
  2. ✅ Cek keberadaan file server/.env
  3. ✅ npm install di semua 4 proyek Node.js (server, client, admin_client, distributor_client)
  4. ✅ flutter pub get di semua 3 aplikasi mobile (mobile, mobile_buyer, mobile_driver)
  5. ✅ npx prisma db push — sinkronisasi skema ke PostgreSQL
  6. ✅ npx prisma generate — generate Prisma Client
  7. ✅ node prisma/seed.js — isi akun demo dan produk contoh
  8. ✅ npm run seed:menus — isi menu navigasi dashboard
  9. ✅ npm run seed:ppob — isi data produk PPOB

Estimasi waktu: 5–15 menit (tergantung kecepatan internet untuk download dependencies).

Double-click start_all.bat untuk membuka 4 jendela terminal sekaligus:

Double-click start_server.bat jika hanya ingin menjalankan backend server (untuk testing API atau menjalankan mobile app saja).

Server berjalan di: http://localhost:4000

5. Menjalankan Aplikasi Mobile (Flutter)

Pastikan backend server sudah berjalan sebelum menjalankan aplikasi mobile.

Konfigurasi Endpoint API

Edit file lib/config/api_config.dart di setiap folder mobile sesuai cara penggunaan:

Opsi A — Emulator Android (Android Studio): Tidak perlu konfigurasi. App otomatis menggunakan http://10.0.2.2:4000.

Opsi B — HP Fisik (WiFi yang sama):

  1. Cek IP laptop: Buka CMD → ketik ipconfig → cari IPv4 Address
  2. Edit di setiap folder mobile:
// lib/config/api_config.dart
static const String _localIp = '192.168.1.8'; // IP Laptop Anda
⚠️ Tambahkan Inbound Rule di Windows Firewall untuk port 4000 jika menggunakan HP fisik.

📱 Menjalankan Setiap App

# Rana Seller (Kasir Merchant)
cd mobile
flutter run

# Rana Buyer (Marketplace Pembeli)
cd mobile_buyer
flutter run

# Rana Driver (Kurir)
cd mobile_driver
flutter run

7. Alur Kerja Harian (Setelah Instalasi)

📋 Setup pertama kali (sekali saja):
  1. Edit server/.env → sesuaikan password PostgreSQL
  2. Buat database: CREATE DATABASE rana_pos; di pgAdmin
  3. Double-click Install manual awal.bat → tunggu selesai
🔁 Setiap hari mau development:
  1. Double-click start_all.bat → buka semua layanan
  2. Buka browser di http://localhost:5173 (Dashboard Merchant)
  3. Atau jalankan flutter run di folder aplikasi mobile yang ingin dikerjakan
🔄 Setelah pull perubahan dari Git (jika ada update schema.prisma):
  1. Double-click update_db.bat → sinkronisasi schema
  2. Restart server

8. Troubleshooting

❌ Error: "Cannot find module 'mongoose'"

Penyebab: File server/src/models/contact.js mengandung kode Mongoose (untuk MongoDB) yang tersisa dari versi lama. Proyek ini 100% menggunakan PostgreSQL + Prisma.

Solusi: Jangan install mongoose. Kode sudah diperbaiki. Jalankan:

cd server
npx prisma db push
npx prisma generate
⚠️ Jangan jalankan npm install mongoose — itu bukan solusinya.

❌ Error: "Authentication failed" (P1000) — Database

Penyebab: Password di server/.env salah atau ada prefix PASSWORD_ yang tersisa.

# ❌ SALAH
DATABASE_URL="postgresql://postgres:PASSWORD_simpledark67@..."

# ✅ BENAR
DATABASE_URL="postgresql://postgres:simpledark67@..."

❌ Error: "npx cannot be loaded" / "running scripts is disabled" (PowerShell)

Penyebab: Windows PowerShell memblokir eksekusi script npm/npx karena Execution Policy = Restricted.

Solusi — Pilih salah satu:

Opsi A (Termudah): Gunakan CMD bukan PowerShell.

Win + R → ketik "cmd" → Enter
cd C:\Riyan\projek\rana-main\server
npx prisma db push

Opsi B (Permanen): Buka PowerShell sebagai Administrator, jalankan:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Ketik Y lalu Enter. Tutup dan buka kembali terminal.

💡 RemoteSigned aman — script lokal (npm, npx) bisa berjalan bebas, script dari internet tetap butuh verifikasi.

Opsi C (Sekali pakai):

powershell -ExecutionPolicy Bypass -Command "npx prisma db push"

❌ Error: "Can't reach database server at localhost"

Solusi:

  • Tekan Win + R → ketik services.msc → cari postgresql-x64-XX → klik Start
  • Atau buka pgAdmin dan pastikan server PostgreSQL terhubung

❌ Error: "The table does not exist" saat jalankan server

Solusi: Schema belum disinkronkan. Double-click update_db.bat atau jalankan:

cd server
npx prisma db push

❌ Error: "Port 4000 is already in use"

Solusi: Ubah PORT=4001 di server/.env, lalu update juga file api_config.dart di semua folder mobile.

❌ Aplikasi Mobile: Network Error / Connection Refused

  • Pastikan server berjalan: akses http://localhost:4000 di browser laptop → harus ada response JSON
  • Jika HP fisik: pastikan IP di api_config.dart sudah benar
  • Tambahkan Inbound Rule di Windows Firewall untuk port 4000
  • Coba akses dari browser HP: http://192.168.x.x:4000

❌ flutter pub get Gagal

Solusi: Jalankan flutter doctor untuk diagnosa. Pastikan Android SDK dan variabel PATH Flutter sudah dikonfigurasi dengan benar.

❌ Install manual awal.bat tidak berjalan / langsung tutup

Penyebab: Script dijalankan dari lokasi yang bukan root folder proyek, atau ada error di awal yang menyebabkan exit.

Solusi: Klik kanan file → Run as administrator. Atau buka CMD, navigasi ke folder proyek, lalu jalankan manual:

cd C:\Riyan\projek\rana-main
"Install manual awal.bat"
