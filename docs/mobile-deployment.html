Panduan Deployment Mobile Full Komplit

Langkah-langkah untuk mem-build dan merilis 3 aplikasi mobile Flutter (Seller, Buyer, Driver) ke Google Play Store.

Versi Dokumen: 1.2.0
Terakhir Diupdate: 2026-03-31

1. Daftar Aplikasi Mobile

Ekosistem Rana memiliki 3 aplikasi mobile terpisah:

  • Rana Seller (POS): Folder mobile/. Digunakan merchant untuk kasir dan stok.
  • Rana Buyer (Marketplace): Folder mobile_buyer/. Digunakan pelanggan untuk belanja.
  • Rana Driver (Delivery): Folder mobile_driver/. Digunakan kurir untuk pengantaran.

2. Persiapan Produksi (Wajib)

A. Konfigurasi URL API

Pastikan aplikasi mengarah ke server produksi Anda. Edit file lib/config/api_config.dart di setiap folder aplikasi:

// Ganti dengan URL domain API Anda
static const String _prodUrl = 'https://api.domainanda.com/api';

B. Update Nama & Package ID

Sesuaikan identitas aplikasi sebelum di-publish:

  • Nama Aplikasi: Edit android/app/src/main/AndroidManifest.xml pada bagian android:label.
  • Package ID: Edit android/app/build.gradle pada bagian applicationId (misal: com.ranamarket.seller).
  • App Icon: Ganti icon di assets/icon/ lalu jalankan: 
    dart run flutter_launcher_icons

3. Penandatanganan Aplikasi (Signing)

Setiap aplikasi membutuhkan Keystore yang berbeda atau sama (tergantung kebijakan Anda) untuk dirilis ke Play Store.

Langkah Membuat Keystore:

# Jalankan di terminal folder aplikasi masing-masing
keytool -genkey -v -keystore upload-keystore.jks -storetype JKS -keyalg RSA -keysize 2048 -validity 10000 -alias upload

Konfigurasi key.properties:

Buat file android/key.properties di setiap folder aplikasi:

storePassword=password_keystore_anda
keyPassword=password_alias_anda
keyAlias=upload
storeFile=../../upload-keystore.jks

4. Proses Build (Android)

Anda bisa memilih build manual via CLI atau otomatis via skrip batch.

Metode A: Skrip Otomatis (Windows Only)

Gunakan file build_apk.bat di root folder untuk mem-build aplikasi Seller dan Buyer secara cepat.

  1. Double click build_apk.bat.
  2. Pilih aplikasi yang ingin di-build (1=Seller, 2=Buyer, 3=Both).
  3. Hasil APK akan otomatis disalin ke folder root/builds/.

Metode B: Manual via Terminal

Jalankan perintah berikut di folder aplikasi masing-masing:

flutter clean
flutter pub get
# Untuk Google Play Store (App Bundle)
flutter build appbundle --release
# Untuk install langsung (APK)
flutter build apk --release

5. Checklist Sebelum Rilis

  • API Config: URL sudah menunjuk ke server produksi HTTPS.
  • Version Code: Sudah dinaikkan di pubspec.yaml (misal: version: 1.0.1+2).
  • Permission: Izin Kamera, Lokasi, dan Storage sudah aktif di AndroidManifest.xml.
  • ProGuard: Pastikan minifyEnabled aktif di build.gradle untuk mengecilkan ukuran APK.

6. Publikasi ke Google Play Console

  1. Buka Google Play Console.
  2. Pilih Create App.
  3. Masuk ke menu Production > Create new release.
  4. Upload file .aab yang ada di folder build/app/outputs/bundle/release/.
  5. Lengkapi Store Listing (Deskripsi, Screenshot, Icon).
  6. Kirim untuk peninjauan (Review).
