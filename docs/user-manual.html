Panduan Operasional Admin

SOP (Standard Operating Procedures) untuk administrator sistem sehari-hari.

Menambah Admin Baru

Untuk saat ini, penambahan Super Admin dilakukan langsung via database atau API Registration (jika diaktifkan).

Via Database:

INSERT INTO "User" (name, email, role, passwordHash) 
VALUES ('Admin Baru', 'admin@rana.com', 'SUPER_ADMIN', 'hashed_password');

Proses Refund Transaksi

  1. Buka Dashboard Admin.
  2. Cari ID Transaksi yang dikeluhkan pelanggan.
  3. Verifikasi status transaksi (harus COMPLETED).
  4. Jika valid, lakukan transfer manual pengembalian dana.
  5. Tandai transaksi sebagai VOID atau REFUNDED di database untuk menjaga akurasi laporan keuangan.

Menangani Laporan Error dari User

Jika ada user melapor "Aplikasi error tidak bisa checkout":

  1. Tanya jam berapa kejadiannya.
  2. Cek log server: pm2 logs --lines 100.
  3. Cari pesan error di sekitar jam tersebut.
  4. Jika error PPOB (Digiflazz), cek saldo deposit di dashboard Digiflazz.

Backup Data Mingguan

Setiap hari Jumat sore, lakukan:

  1. Jalankan script backup database.
  2. Download file hasil backup ke laptop/cloud storage.
  3. Cek ukuran file (pastikan tidak 0 bytes).

Persetujuan Distributor (KYC)

Setiap pendaftar baru di Distributor Portal harus diverifikasi dokumennya oleh Super Admin sebelum bisa berjualan.

  1. Masuk ke Dashboard Admin > Menu Merchants/Distributors.
  2. Cek dokumen NPWP dan izin usaha yang diupload.
  3. Klik tombol Approve untuk mengaktifkan akun mereka.

Manajemen Saldo (Wallet)

Admin dapat melihat saldo seluruh toko dan distributor.

  • Top Up: Verifikasi bukti transfer dari merchant, lalu tambahkan saldo secara manual di dashboard admin.
  • Withdrawal: Proses permintaan penarikan dana dari merchant/distributor maksimal 1x24 jam kerja.
